# SKILL.md - openviking-search

## Описание
Семантический поиск по контексту пользователя через OpenViking. Позволяет находить релевантную информацию из MEMORY.md, daily notes, SKILL.md файлов по смыслу, а не по ключевым словам.

## Когда использовать
- Пользователь спрашивает о прошлых событиях, решениях, настройках
- Нужно найти информацию из памяти без ручного чтения файлов
- Требуется персонализированный ответ с цитированием источников

## Что НЕ делает
- Не заменяет чтение файлов напрямую
- Не хранит оригинальные данные — только индексирует
- Не меняет текущую архитектуру памяти

---

## API

### Поиск контекста

```python
from skills.openviking-search.skill import search_context

results = search_context("как настроить ollama", limit=5)
# Returns: [{"source": "memory/2026-03-16.md", "content": "...", "score": 0.89}]
```

### Параметры

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `query` | str | - | Запрос для поиска |
| `limit` | int | 5 | Максимум результатов |
| `score_threshold` | float | 0.5 | Минимальный порог релевантности |
| `target_uri` | str | "" | Ограничить поиск (например "viking://resources/") |

---

## Пример использования

```python
# Поиск информации о настройках
context = search_context("какие модели ollama установлены")

for item in context:
    print(f"{item['source']} (score: {item['score']:.2f}):")
    print(item['content'][:200])
    print("---")
```

## Архитектура

```
Запрос пользователя
      ↓
[search_context()]
      ↓
OpenViking.find()
      ↓
Векторный поиск в viking.db
      ↓
Результаты с рейтингом
      ↓
Форматирование для вывода
```

---

## Ограничения

- Требуется предварительная индексация файлов
- Первый поиск после индексации может быть медленным
- Для обновления индекса нужно перепустить sync
