# OpenViking Integration — Проект

**Created:** 2026-03-17  
**Status:** In Progress  
**Goal:** Интеграция семантического поиска через OpenViking в OpenClaw

---

## 📋 Обзор

### Цель проекта
Интегрировать OpenViking как семантический движок для поиска по памяти OpenClaw:
- Долгосрочная память (MEMORY.md)
- Дневники (memory/*.md)
- Контекст агента (SOUL.md, USER.md и др.)
- Уроки и ошибки (.learnings/)

### Почему OpenViking
- Файловая парадигма вместо плоского RAG
- L0/L1/L2 уровни контекста (абстракция/обзор/полное)
- Рекурсивный поиск по директориям
- Наблюдаемый поиск (визуализация пути)

### Текущий стек OpenClaw
- **Embedding:** hf:ggml-org/embeddinggema-300m-qat-q8_0-GGUF (384 dims, локальная)
- **Векторная база:** SQLite Vec
- **Поиск:** Гибридный (70% векторный + 30% FTS5)

### OpenViking стек
- **Embedding:** nomic-embed-text (768 dims, через ollama)
- **Векторная база:** LanceDB
- **Поиск:** Семантический + гибридный

## 💡 Анализ автоматизации

### Вариант 1: Настроить openviking-store.js для авто-загрузки
Изменить так, чтобы данные добавлялись в OpenViking без флага `--viking`.

**Плюс:** Простой, работает сразу.  
**Минус:** Не идеально — все данные дублируются.

### Вариант 2: Создать триггер для "вспомни"
Настроить OpenClaw так, чтобы при паттернах "вспомни", "что я говорил про" автоматически вызывался `openviking-agent.js`.

**Плюс:** Чётко, прозрачно.  
**Минус:** Требует настройки (нет готового решения).

### Вариант 3: Авто-дополнение запроса к модели
Перед отправкой запроса к модели:
1. Проверить паттерны ("вспомни", "из памяти")
2. Вызвать `openviking-agent.js`
3. Добавить результат в контекст запроса

**Плюс:** Идеально, прозрачно.  
**Минус:** Требует модификации OpenClaw.

### Вариант 4: Использовать встроенные механизмы OpenViking
Проверить документацию OpenViking на наличие встроенных механизмов интеграции.

**Плюс:** Официально, стабильно.  
**Минус:** Требует изучения.
