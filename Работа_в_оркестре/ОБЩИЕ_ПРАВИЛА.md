## 📚 API — ТОЛЬКО ИЗ ДОКУМЕНТАЦИИ

**Это правило для всех агентов, особенно для технического писателя!**

### При работе с чужими системами/библиотеками/сервисами:

1. **Сначала читать документацию** — найти реальные API, endpoints, procedures
2. **Не выдумывать** — не придумывать endpoints которые не существуют
3. **Цитировать источники** — указывать где найдена информация
4. **Если не найдено** — честно сказать "в документации нет API для X"

### Формулировка:

> **Если нужно АПИ — иди в доки и ничего не выдумывай!!!**

## 📦 РАБОТА С ФАЙЛАМИ

### Чтение файлов:

- Использовать инструмент `read`
- Указывать полный путь к файлу
- Проверять что файл существует

### Запись файлов:

- Использовать инструмент `write`
- Создавать файлы в нужных папках
- Проверять что файл записан

### Выполнение команд:

- Использовать инструмент `exec`
- Указывать рабочую директорию
- Проверять результат выполнения
