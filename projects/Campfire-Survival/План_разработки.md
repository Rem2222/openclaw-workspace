# План разработки — Campfire Survival

## Общая информация
- **Проект:** Campfire Survival
- **Тип:** 2D HTML5 Game
- **Масштаб:** Маленький (MVP)
- **Технологии:** Phaser 3, Vite, JavaScript
- **Оценка времени:** ~2 часа

---

## Этапы разработки

### Этап 1: Планирование (ТЕКУЩИЙ) 📋
**Цель:** Декомпозиция задачи, определение структуры проекта  
**Результат:** План_разработки.md

**Что делаем:**
- Определяем этапы разработки
- Оцениваем трудозатраты
- Задаём структуру проекта

---

### Этап 2: Техническое задание 📝
**Цель:** Детальное описание реализации  
**Результат:** ТЗ.md

**Что описываем:**
- Структура проекта (файлы, папки)
- Классы и модули (GameScene, Player, Tree, Log, Monster, Campfire)
- Игровой цикл и состояния
- Процедурная генерация графики
- Формулы (уменьшение огня, спавн монстров, радиус света)
- UI элементы и их логика

**Примечание:** Для игр ТЗ можно сделать кратким — вся игровая логика уже описана в SPEC.md

---

### Этап 3: Разработка 🎮
**Цель:** Реализация игры  
**Результат:** Готовый код игры

**Что реализуем:**
- ✅ Инициализация Phaser + Vite
- ✅ GameScene — основной сценарий игры
- ✅ Player — движение WASD, сбор логов, доставка к костру
- ✅ Tree — процедурная отрисовка, рубка
- ✅ Log — отрисовка, перенос игроком
- ✅ Campfire — уменьшение burn level, динамический свет
- ✅ Monster — спавн в темноте, AI (движение к игроку)
- ✅ Game Over — экран проигрыша, рестарт (R)
- ✅ UI — Burn meter, Survival timer, Carrying indicator

**Важно:** Для 2D игры разработка — ОДИН этап. Не разделяем на backend/frontend как веб-приложение.

---

### Этап 4: Тестирование 🧪
**Цель:** Ручное тестирование  
**Результат:** Рабочая игра без критических багов

**Что проверяем:**
- Движение игрока
- Рубка деревьев
- Сбор и переноска логов
- Восстановление костра
- Спавн монстров
- Game Over при касании монстра
- Рестарт игры

---

### Этап 5: Документация 📖
**Цель:** README.md  
**Результат:** Описание проекта

**Что пишем:**
- Описание игры
- Как запустить
- Управление
- Скриншоты (опционально)

---

```
Этап 1: Планирование
       ↓
```
