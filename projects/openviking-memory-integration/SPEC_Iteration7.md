# SPEC_Iteration7.md — Веб-интерфейс для OpenViking Memory

## 📋 Обзор

**Итерация 7** — создание веб-интерфейса для просмотра и навигации по семантической памяти OpenViking.

**Цель:** Предоставить интуитивный веб-интерфейс для исследования, поиска и чтения сохранённой информации в OpenViking.

**Статус:** MVP — минимально жизнеспособный продукт для валидации концепции.

---

## 🎯 Видение

Создать простой, но функциональный веб-интерфейс, который позволяет:

1. **Искать** информацию по ключевым словам с фильтрами
2. **Навигировать** по иерархии ресурсов
3. **Читать** контент на разных уровнях детализации (L0/L1/L2)
4. **Взаимодействовать** с памятью в интерактивном режиме

Это мост между сырым API OpenViking и человеческим восприятием — превращает текстовые URI и JSON в понятный интерфейс.

---

## 🎯 Цели

### Основная цель

Создать веб-интерфейс, который позволяет пользователю легко исследовать и читать информацию из OpenViking без знания API.

### Критерии успеха

| Критерий | Описание | Как измерить |
|----------|----------|--------------|
| **Функциональность** | Все основные функции работают | Чек-лист функциональных требований |
| **Простота** | Интерфейс интуитивен | Пользователь может найти информацию без инструкций |
| **Скорость** | Поиск и чтение быстры | < 2 сек на поиск, < 1 сек на чтение L0 |
| **Стабильность** | Нет критических ошибок | 0 падений сервиса при正常使用 |

## 📝 Функциональные требования

### FR-1: Поиск с фильтрами

**Описание:** Пользователь может искать информацию с различными фильтрами.

**Функции:**

| Фильтр | Описание | Тип | Значения по умолчанию |
|--------|----------|-----|----------------------|
| **Запрос** | Ключевые слова для поиска | Text | - |
| **Дата от** | Минимальная дата ресурса | Date | - |
| **Дата до** | Максимальная дата ресурса | Date | Сегодня |
| **Тег/Категория** | Фильтр по теме | Text | Все |
| **Шаблон** | Фильтр по шаблону имени | Regex | - |

**Сценарий использования:**

```
Пользователь вводит "интеграция" в поле поиска.
Фильтрует по дате "с 2026-03-01".
Жмёт "Найти".
Получает список релевантных ресурсов с датами и краткими описаниями.
```

**Приоритет:** High

---

### FR-2: Просмотр уровней L0/L1/L2 во вкладках

**Описание:** При выборе ресурса пользователь видит контент на трёх уровнях детализации во вкладках.

**Уровни:**

| Уровень | Имя вкладки | Описание | Время загрузки |
|---------|-------------|----------|----------------|
| **L0** | Abstract | Краткое резюме (~100 токенов) | < 500ms |
| **L1** | Overview | Обзор с навигацией (~2000 токенов) | < 1s |
| **L2** | Full | Полный контент | < 2s |

**Сценарий использования:**

```
Пользователь кликает на ресурс в результатах поиска.
Открывается панель просмотра с вкладками Abstract/Overview/Full.
По умолчанию открыта вкладка Abstract (самая быстрая).
Пользователь переключается на Overview для более детальной информации.
При необходимости переключается на Full для полного текста.
```

**Техническое требование:** Переключение между вкладками не должно перезагружать страницу.

**Приоритет:** High

---

### FR-3: Дерево ресурсов с кликабельными ссылками

**Описание:** Пользователь может видеть иерархию ресурсов в виде дерева.

**Функции:**

- Отображение папок и файлов в виде вложенного дерева
- Возможность сворачивать/разворачивать ветки
- Кликабельные имена файлов открывают ресурс
- Иконки для папок и файлов

**Пример структуры:**

```
viking://resources/
├── memory-abc123/
│   ├── memory-abc123.md  ← клик открывает L2
│   └── .context/
│       ├── abstract.txt
│       └── overview.txt
├── memory-def456/
│   └── memory-def456.md
└── sessions/
    └── session-2026-03-17/
        └── session.md
```

**Сценарий использования:**

```
Пользователь открывает раздел "Ресурсы".
Видит дерево всех ресурсов.
Разворачивает папку "memory-abc123".
Кликает на "memory-abc123.md".
Открывается просмотрщик с вкладками L0/L1/L2.
```

**Приоритет:** Medium

---

### FR-4: Интерактивный режим

**Описание:** Чат-интерфейс для взаимодействия с памятью.

**Функции:**

- Ввод вопросов на естественном языке
- Автоматический поиск релевантной информации
- Формирование ответа с ссылками на источники

**Сценарий использования:**

```
Пользователь вводит: "Что мы обсуждали про интеграцию OpenViking?"
Система ищет релевантную информацию.
Возвращает ответ: "Вы обсуждали использование OpenViking как семантической 
памяти для OpenClaw. Были созданы скрипты ov_store, ov_search, ov_read."
Ссылки на источники: [memory-abc123], [memory-def456]
```

**Приоритет:** Medium (расширение после MVP)

---
