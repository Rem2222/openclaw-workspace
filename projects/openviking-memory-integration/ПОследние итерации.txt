 ### Почему Unauthorized

 Console Web UI требует авторизации, потому что:

 1. VikingBot gateway запустил защищённый интерфейс
 2. Нужен токен/пароль для доступа
 3. По умолчанию нет публичного доступа

 ### Как авторизоваться

 #### Вариант 1: Через API ключ

 1. Найти API ключ в конфигурации:
 ```bash
   cat ~/.openviking/ov.conf | grep -A 5 "console"
 ```

 2. Использовать в заголовке:
 ```bash
   curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:18791/
 ```

 #### Вариант 2: Прямой доступ через Python

 ```python
   import openviking as ov

   client = ov.OpenViking(path="~/.openviking/workspace")
   client.initialize()

   # Список ресурсов
   resources = client.ls("viking://resources/")

   # Семантический поиск
   results = client.search("запрос", limit=5)

   # Чтение контента
   content = client.read("viking://resources/uri")
 ```

 ### Вариант 2: Использовать Python API напрямую

 Создать файл в workspace OpenViking использовать local://:

 ```python
   import openviking as ov                                                                                                                   
   # 1. Создаём файл в workspace
   with open("/home/rem/.openviking/workspace/viking/session-xxx.md", "w") as f:
       f.write(content)

   # 2. Добавляем ресурс
   client = ov.OpenViking(path="/home/rem/.openviking/workspace")
   client.initialize()
   result = client.add_resource(path="local://session-xxx.md", description="...")
 ```

 Плюсы:                                                                                                                                       - Не требует VikingBot
 - Работает локально

 Минусы:
 - Нужно разбираться с filesystem OpenViking                                                                                                  - Может быть медленее
