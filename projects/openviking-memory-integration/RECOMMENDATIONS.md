# Рекомендации по улучшению OpenViking Memory Integration

**Дата создания:** 2026-03-17  
**Статус:** Открыт для работы

---

## 1. Настроить регулярный бэкап

### Почему это важно:
- Автоматическое сохранение контекста сессий
- Защита от потери данных
- Архивирование исторической информации

### Варианты реализации:

#### Вариант A: Cron (системный)
```bash
# Каждую ночь в 23:00
0 23 * * * ~/.openclaw/skills/openviking-memory/scripts/auto_save >> /var/log/openviking-backup.log 2>&1
```

#### Вариант B: "Спокойной ночи" (через OpenClaw)
```bash
# Добавить в HEARTBEAT.md секцию "Вечер"
```

### Как часто:
- **Ежедневно** — для сохранений сессий
- **Еженедельно** — для полного архива

### Приоритет: **Высокий** ⭐⭐⭐

## 2. Установить модель qwen2.5:7b

### Почему это важно:
- Сейчас генерация L0/L1 (abstract/overview) не работает
- Возвращается шаблон вместо реального контента
- Без модели семантический поиск менее эффективен

### Как установить:
```bash
# Через Ollama
ollama pull qwen2.5:7b

# Проверить
ollama list
```

### Альтернативы:
- `qwen2.5:1.5b` — уже установлена, но медленная
- `doubao-seed-2-0-pro-260215` — рекомендована OpenViking, но платная (Volcengine)

### Приоритет: **Высокий** ⭐⭐⭐

## 3. Интегрировать с daily notes

### Почему это важно:
- Автоматическое сохранение контекста сессий
- Упрощение workflow
- Единое место для хранения

### Что нужно сделать:
1. Добавить скрипт в HEARTBEAT.md
2. Настроить автоматическое сохранение при каждом heartbeat
3. Добавить проверку дубликатов

### Пример:
```bash
# В HEARTBEAT.md
## При каждом heartbeat (если прошло >1 часа)
~/.openclaw/skills/openviking-memory/scripts/auto_save
```

### Приоритет: **Средний** ⭐⭐

## 4. Добавить метрики

### Почему это важно:
- Понимание использования памяти
- Оптимизация запросов
- Отслеживание здоровья системы

### Какие метрики:
- Количество запросов к памяти
- Время отклика
- Загруженность памяти
- Наиболее частые запросы

### Пример дашборда:
```bash
# Скрипт для просмотра статистики
ov_stats

# Вывод:
Total resources: 127
Searches today: 15
Avg response time: 2.3s
Memory usage: 45%
```

### Приоритет: **Низкий** ⭐

## 5. Добавить дашборд OpenViking

### Почему это важно:
- Визуализация загрузки памяти
- Мониторинг здоровья системы
- Удобный интерфейс для управления

### Что есть в оригинальном OpenViking:
- Web-интерфейс для просмотра ресурсов
- Статистика использования
- Управление памятью

### Как запустить:
```bash
# Через VikingBot gateway
vikingbot gateway

# Или через отдельный сервер
vikingbot web
```
