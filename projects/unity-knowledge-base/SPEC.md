# SPEC.md — Единое хранилище знаний

## Название проекта

**Unity Knowledge Base** — унифицированная система хранения и быстрого поиска информации

---

## 1. Контекст и проблема

### Текущая ситуация

Данные разрознены:

| Тип | Где хранится | Проблема |
|-----|--------------|----------|
| **Документы** | Файловая система, Google Docs | Медленный поиск |
| **Конфиги** | `~/.openclaw/`, проекты | Разбросаны |
| **Код** | Git репозитории | Нужен grep |
| **Заметки** | MEMORY.md, daily notes | Ручной поиск |
| **Сессии** | JSONL файлы | Не индексированы |

### Проблема

LCM (Lossless-Claw) работает только для новых сессий. Исторические данные (до 23 марта) не захвачены.

Модель не имеет быстрого доступа к информации — приходится искать по файловой системе.

---

## 2. Цель проекта

**Создать единую систему**, которая:
- Индексирует ВСЕ типы данных (документы, конфиги, код, заметки, историю)
- Обеспечивает быстрый семантический поиск
- Подключается к LLM для augment generation
- Автоматически обновляет индекс при изменениях

---

## 3. Что нужно хранить

### 3.1 Документы
- `.md` — Markdown файлы (README, заметки)
- `.txt` — Текстовые файлы
- `.pdf` — PDF документы (опционально)
- `.yaml/.yml` — YAML конфиги
- `.json` — JSON файлы

### 3.2 Конфигурации
- `~/.openclaw/` — OpenClaw config, skills, workspace
- Проекты 1С — конфигурации, обработки, отчёты
- Системные конфиги

### 3.3 Код
- Исходники проектов
- Скрипты
- 1С код (epf, erf, xml)

### 3.4 Заметки и память
- `MEMORY.md` — долгосрочная память
- `memory/*.md` — ежедневные заметки
- `SOUL.md`, `USER.md`, `AGENTS.md`

### 3.5 История сессий
- JSONL файлы сессий
- LCM данные (для бэкфила)

## 4. Архитектура решения

### 4.1 Компоненты

```
┌─────────────────────────────────────────────────────────────┐
│                    Unity Knowledge Base                      │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐│
│  │  Indexer    │  │  Vector DB  │  │  Query Engine       ││
│  │  (Python)   │→ │  (Qdrant)   │← │  (API + Skills)     ││
│  └─────────────┘  └─────────────┘  └─────────────────────┘│
│         ↑                                    ↓              │
│  ┌─────────────┐                      ┌─────────────────┐  │
│  │ File Watcher│                      │  OpenClaw       │  │
│  │ (auto-sync) │                      │  (LLM + Tools)  │  │
│  └─────────────┘                      └─────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 Stack

| Компонент | Технология |
|-----------|------------|
| **Vector DB** | Qdrant (Docker) |
| **Embeddings** | local model ( sentence-transformers ) |
| **Indexer** | Python + watchdog |
| **API** | FastAPI |
| **OpenClaw Skill** | unified-memory (обновлённый) |

### 4.3 Источники данных для индексации

```

├── workspace/
│   ├── memory/          # Ежедневные заметки
│   ├── MEMORY.md       # Долгосрочная память
│   ├── SOUL.md         # Личность
│   ├── USER.md         # О пользователе
│   └── projects/       # Проекты
│       ├── OpenMOSS/
│       └── ...
├── skills/             # Скиллы
└── agents/             # JSONL сессии

~/.openclaw/workspace/projects/  # Проекты Романа
```

## 6. Критерии приёмки

### 6.1 Работает поиск
```
Пользователь: "?знания i18n OpenMOSS"
Ответ: Найдены документы о i18n для OpenMOSS (18-20 марта)
```

### 6.2 Индексация завершена
- Все .md файлы из workspace проиндексированы
- MEMORY.md доступен через поиск
- Исторические сессии (март 19-23) доступны

### 6.3 Скорость
- Поиск < 500ms
- Индексация нового файла < 5 сек
