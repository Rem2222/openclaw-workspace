# ТЗ — Добавление z.ai провайдера в CodexBar-Win

## 1. Введение

CodexBar-Win — настольное Python-приложение (~1936 строк, `codexbar.py`) для мониторинга использования LLM-провайдеров. На данный момент поддерживает Claude и OpenAI. Задача — добавить поддержку z.ai (GLM-провайдер) в качестве третьего провайдера.

**z.ai API:**
- Endpoint: `GET https://api.z.ai/api/monitor/usage/quota/limit`
- Auth: `Authorization: Bearer {token}` (формат токена: `tp-...`)
- Response: `{"quota": N, "used": N, "remaining": N, "reset_date": "..."}`

**Архитектура приложения:**
- `CodexBarPopup` — главный класс UI (customtkinter)
- `ClaudeDataFetcher` — класс для получения данных, 4 стратегии (CLI, OAuth, Cookies, JSONL)
- Tabs: Claude, OpenAI → добавляется z.ai
- Config: `~/.codexbar/config.json`

## 2. Функциональные требования

### FR-1: Получение данных z.ai
- Приложение должно выполнять GET-запрос к `https://api.z.ai/api/monitor/usage/quota/limit`
- Авторизация через Bearer token
- Парсинг ответа: `quota`, `used`, `remaining`, `reset_date`
- Периодическое автообновление (по таймеру, как у других провайдеров)

### FR-2: Отображение данных
- Отдельная вкладка «z.ai» в UI
- Отображение: quota, used, remaining, reset_date
- Прогресс-бар: used / quota
- Цветовая индикация: зелёный (< 70%), жёлтый (70-90%), красный (> 90%)

### FR-3: Настройки
- Поле ввода API token (masked) в окне настроек
- Кнопка «Test» для проверки токена
- Сохранение токена в `~/.codexbar/config.json`
- Поддержка env var `ZAI_API_TOKEN` как альтернативного источника

### FR-4: Обработка ошибок
- Таймаут запроса (10 сек)
- Невалидный токен → понятное сообщение об ошибке
- Нет сети → индикация в UI без краша
- Ошибки z.ai не влияют на работу Claude/OpenAI табов

## 3. Нефункциональные требования

### NFR-1: Производительность
- Запрос к z.ai API — не более 10 секунд таймаут
- UI не блокируется во время запроса (async/threaded, как у других fetcher'ов)
- Добавление ~150-200 строк кода

### NFR-2: Безопасность
- Токен хранится в config.json (по аналогии с другими провайдерами)
- Токен маскируется в UI (password field)
- Токен не логируется в открытом виде

### NFR-3: Совместимость
- Windows 10/11 (целевая платформа CodexBar-Win)
- Python 3.10+
- Не ломать существующий функционал Claude/OpenAI

### NFR-4: Поддерживаемость
- Код по аналогии с существующими fetcher'ами
- Единый файл `codexbar.py` (пока без рефакторинга в модули)

### Этап 2: Класс ZaiDataFetcher

**Цель:** Реализовать fetcher для z.ai API по аналогии с `ClaudeDataFetcher`.

**Задачи:**
1. Создать класс `ZaiDataFetcher` по аналогии с `ClaudeDataFetcher`
2. Реализовать метод `_fetch_from_api()`:
   ```
   GET https://api.z.ai/api/monitor/usage/quota/limit
   Authorization: Bearer {token}
   → {"quota": N, "used": N, "remaining": N, "reset_date": "..."}
   ```
3. Реализовать чтение API token: env var `ZAI_API_TOKEN` → fallback в config.json
4. Обработка ошибок:
   - `requests.Timeout` → вернуть ошибку с сообщением «Таймаут z.ai API»
   - `requests.ConnectionError` → «Нет подключения к z.ai»
   - HTTP 401/403 → «Невалидный z.ai API token»
   - Некорректный JSON → «Ошибка парсинга ответа z.ai»
5. Нормализация данных в единый формат (dict с ключами quota, used, remaining, reset_date)

**Критерии приёмки:**
- [ ] Класс `ZaiDataFetcher` создан в `codexbar.py`
- [ ] Метод `_fetch_from_api()` корректно выполняет GET-запрос к z.ai API
- [ ] Token читается из env var или config.json
- [ ] Все типы ошибок обрабатываются без краша приложения
- [ ] Возвращаемые данные нормализованы (dict с quota/used/remaining/reset_date)
- [ ] Код следует стилю существующих fetcher'ов

## 6. Риски

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| API z.ai меняет формат ответа | Низкая | Среднее | Гибкий парсер с fallback-полями |
| Единый файл ~2100 строк становится неудобным | Среднее | Низкое | Задокументировать для будущего рефакторинга в модули |
| Токен z.ai истекает/ротируется | Низкая | Низкое | Кнопка Test в настройках, понятная ошибка в UI |
| z.ai API недоступен из-за GeoIP/VPN | Низкая | Среднее | Понятное сообщение об ошибке, retry |
| Несовместимость с Python < 3.10 | Низкая | Низкое | Требование 3.10+ документировано |
