# REM-94: Мульти-провайдер виджет

## Концепция

Компактный виджет-панель, показывающий квоты всех подключённых провайдеров одновременно. Взгляд — минималистичный, информативный, с акцентом на быстрое считывание. Размер виджета остаётся ~220x220, но внутри меняется компоновка: вместо одного большого круга — сетка 2×3 с ячейками провайдеров.

- Процент: 11px, белый, bold

## Компоновка (Layout)

### Сетка 2×3

```
┌─────────────────────────────────┐
│ [CL●] [OA●] [ZA●]              │  ← ячейки провайдеров, сверху цветная полоска
│  ████░░  15%   ███████░ 45%    │
│ Resets   Resets                 │
├─────────────────────────────────┤
│ [MM●] [OC●] [OL●]              │
│  ██████░░ 67%  ░░░░░░░  0%    │
│ Resets   Resets                 │
└─────────────────────────────────┘
```

- Всего 6 ячеек (2 ряда × 3 колонки)
- Неактивные провайдеры (нет данных) — затемнены, opacity 40%
- Активный провайдер (текущая вкладка) — подсвечен бордером `#6366F180`
- Размер виджета: **260×200 px** (вместо 220×220 — чуть шире для 3 колонок)

### Ячейка провайдера

- Верхняя цветная полоска: 3px, accent color
- Иконка/индикатор: 8px круг цветного
- Название: 2 буквы (CL, OA, ZA, MM, OC, OL)
- Progress bar: высота 6px, скругление 3px
- Процент: под баром
- Reset: под процентом, мелкий серый

## Данные

### Показываемые данные (только 5h лимиты)

Для каждого провайдера:
1. `session_used_pct` — session quota percentage (основной лимит)
2. `session_reset` — время до ресета (если есть)

### Индикатор состояния

- Зелёный ● — данные свежие (< 5 мин)
- Жёлтый ● — данные старые (5–15 мин)
- Серый ● — данных нет / провайдер не подключен

## Взаимодействия

| Действие | Результат |
|---|---|
| ЛКМ (drag) | Перетащить виджет |
| ЛКМ (click, < 5px) | Цикл прозрачности (0.25 → 0.50 → 0.75 → 1.0) |
| ПКМ | Toggle click-through режим |
| Hover на ячейке | Подсветка ячейки |

### Контекстное меню (ПКМ)

- ✅ Click-through ON/OFF
- ❌ Close

## IPC формат

Данные передаются через `codexbar_widget.txt`:

```
pct|provider|wp|all_providers_json
```

Пример:
```
15|claude|0|{"claude":{"pct":15,"reset":"Resets 4pm"},"openai":{"pct":45,"reset":"Resets 6pm"},"zai":{"pct":0,"reset":"—"},...}
```

- `pct` — session % активного провайдера (для ring)
- `provider` — текущий провайдер
- `wp` — weekly % (не используется в мульти-режиме, 0)
- `all_providers_json` — JSON со всеми провайдерами для панели

## Алгоритм

### 1. Каждые 5 мин (как сейчас):
codexbar.py фетчит данные всех 6 провайдеров и формирует JSON для всех.

### 2. При смене вкладки:
`_on_tab_switch` вызывает `_write_data` с полным JSON всех провайдеров.

### 3. premium_widget.py читает файл:
```python
parts = data.split("|")
pct = int(parts[0])           # ring %
active = parts[1]             # active provider
providers_data = json.loads(parts[2])  # all providers
```

### 4. Отрисовка:
- Рисуем 6 ячеек в сетке 2×3
- Каждая ячейка: цветная полоска + имя + progress bar + процент + reset
- Активная ячейка подсвечена

## Фичи (включены по умолчанию)

- Показывает всех 6 провайдеров
- Цветовая маркировка
- Progress bar для каждого
- Click-through режим
- Цикл прозрачности по клику
- Drag-and-drop
- Сохранение позиции между рестартами
- Сохранение состояния (opacity, click-through) между рестартами

## Фичи (отключены / будущие)

- Ollama endpoint настройка
- provider visibility фильтры (REM-45)
