## 6. БЕЗОПАСНОСТЬ

### 6.1 7 уязвимостей агентов

**Источник:** Pavel Stupko, 18.02.2026  
**Статистика:** 341 вредоносный скилл на ClawHub, 135 000 серверов торчат наружу

| # | Уязвимость | Решение |
|---|-----------|---------|
| 0 | **Скилл-инъекции** | Читайте текст скилла, не описание. Добавьте правило в claude.md проверять скиллы перед установкой |
| 1 | **Сервер торчит наружу** | `ufw status` — закрой всё кроме SSH и нужного |
| 2 | **Ключи в открытом виде** | Вынеси секреты в Vault или .env вне рабочей директории. Никогда не давай агенту доступ к ~/.ssh |
| 3 | **Утечка данных** | Output DLP: regex + entropy-анализ на каждый ответ |
| 4 | **Промпт-инъекция через веб** | Заведи белый список источников, аппрувь новые сайты вручную |
| 5 | **Сжигание токенов** | Жёсткий лимит ($5-10 на старте). Логируй расход |
| 6 | **TOS-бан** | Используй API, не подписку для автоматических задач. Заведи отдельную учётку для агента |

## 8. ЭВОЛЮЦИЯ ПОДХОДА

### 8.1 Vibe Coding → Agentic Engineering

**Источник:** Andrej Karpathy, февраль 2026

**Почему вайбкодинг уходит в прошлое:**

| Vibe Coding (2025) | Agentic Engineering (2026) |
|-------------------|---------------------------|
| Модели программировали нестабильно | Агенты кодят часами |
| Скорее развлечение | Профессиональный навык |
| Промпт с «все-все-все» | Настройка и поддержка агентов |
| Одноразовые промпты | Личность агента, постоянная память |
| 20-30% кода через ИИ | **80% кода через ИИ** (Karpathy) |

**Промпт-инжиниринг устаревает:**
- Агенты хранят контекст долго
- Способны планировать и разбивать задачи
- Важна настройка агента, а не промптов

**Аналогия:** Настройка агента — как работа с джуном:
1. Описываешь задачу
2. Видишь идеи по реализации
3. Даёшь фидбек
4. Получаешь результат
