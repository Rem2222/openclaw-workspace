# Справочник терминов

**Для быстрого понимания терминологии вайбкодинга и agentic engineering**

---

## А

**Agentic Engineering** — подход к разработке через управление ИИ-агентами. Агенты кодят часами, выдавая качественный результат. Karpathy пишет 80% кода через агенты.

**AI-агент** — автономная система, способная планировать задачи, разбивать на этапы и выполнять их.

**Analyst (Аналитик)** — субагент для анализа требований и проектирования решений.

---

## Б

**BSL (Basis Scripting Language)** — язык программирования 1С:Предприятие.

**БСП (Библиотека Стандартных Подсистем)** — набор типовых решений для 1С от сообщества.

---

## В

**Vibe Coding (ВайбКодинг)** — подход к программированию через естественный язык. Термин ввёл Andrej Karpathy в феврале 2025.

---

## Г

**GraphRAG** — графовый RAG для связанных данных. Использует графовые БД (Neo4j) для поиска по связям.

---

## Д

**Documentation as Code** — подход к документации, когда она хранится в репозиториях и обрабатывается как код.

---

## К

**Claude Code** — агент от Anthropic для разработки кода.

**Chunk (Чанк)** — фрагмент текста для векторизации в RAG.

**CRAG** — критический RAG с проверкой фактов перед выдачей ответа.

**Cursor** — редактор кода с интегрированным ИИ. Основной инструмент для вайбкодинга.

**Custom Modes** — пользовательские режимы в Cursor с разными моделями и настройками.

---

## L

**LLM (Large Language Model)** — большая языковая модель (GPT, Claude, Gemini и др.).

**LangGraph** — Python-фреймворк для разработки ИИ-агентов.

**LightRAG** — лёгкий RAG для быстрых ответов.

---

## M

**MCP (Model Context Protocol)** — протокол для подключения LLM к инструментам и данным.

**Memory Bank** — система хранения долгосрочного контекста для ИИ.

**Milvus** — масштабируемая векторная база данных.

---

## O

**OpenSpec** — фреймворк для спецификаций в SDD.

**Opus** — флагманская модель от Anthropic.

---

## P

**PRD (Product Requirements Document)** — документ с требованиями к продукту.

**Prompt Engineering (Промпт-инжиниринг)** — искусство написания промптов. Устаревает в пользу настройки агентов.

**pgVector** — расширение PostgreSQL для векторного поиска.

---

## Q

**Qdrant** — векторная база данных с фильтрацией.

---

## R

**RAG (Retrieval-Augmented Generation)** — подключение ИИ к внешним данным через поиск.

**RAPTOR** — иерархический RAG для больших данных.

**Reranking** — переранжировка результатов поиска в RAG.

**ReAct** — паттерн промптинга (Reason + Act).

---

## S

**SDD (Software Design Document)** — документ с архитектурой проекта.

**Spec-Kit** — фреймворк для спецификаций в SDD.

**Sonnet** — модель от Anthropic, дешевле Opus.

---

## T

**TaskMaster** — фреймворк для управления задачами через агентов.

**TOS (Terms of Service)** — условия использования. Бан за нарушение при автоматизации.

---

## Э

**Embedding** — векторное представление текста для поиска.

**Enterprise** — корпоративный сегмент (большие компании).

---

## Ф

**Фреймворк** — набор инструментов и паттернов для решения задач.

---

## C

**Chunking** — разбиение текста на фрагменты для векторизации.

---

## Ч

**ChromaDB** — простой векторный движок для RAG.

---

## Я

**Yandex AI Studio** — российская облачная платформа для ИИ-агентов.

---
