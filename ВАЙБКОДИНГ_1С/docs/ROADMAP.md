**Цель:** Освоить MCP-серверы, субагенты, фреймворки

### 📦 MCP-серверы

**Что это:** Model Context Protocol — способ подключать LLM к инструментам и данным 1С

**8 серверов от vibecoding1c.ru:**

| # | Сервер | Для чего |
|---|--------|----------|
| 1 | Поиск по коду (RAG+Fulltext) | Поиск по конфигурации |
| 2 | Поиск по справке (RAG+Fulltext) | Справка и запросы |
| 3 | Поиск по шаблонам (RAG) | Шаблоны кода + свои |
| 4 | Проверка синтаксиса BSL LS | BSL Language Server |
| 5 | Проверка кода (1С:Напарник) | Нужен ключ |
| 6 | Поиск по метаданным (Граф + Субагент) | Бизнес-сущности |
| 7 | Поиск по функциям БСП (RAG+Fulltext) | Любые версии |
| 8 | Генерация форм (RAG + Субагент) | Для 1С |

**Формат:** Docker контейнеры, подключаются одной строкой  
**Цена:** 7 000 ₽ (опционально, есть бесплатные альтернативы)

**Альтернативы:**
- **1c-mcp-toolkit** (ROCTUP) — REST API для метаданных
- **1c-web-session** (RooLee10) — управление 1С через Playwright
- Каталог: https://benchlabs.ru/mcp-servers (18 серверов)

### 📝 Задачи для практики

#### 1. Генерация кода для 1С

**Модули документов:**
```
"Напиши модуль документа 'Заказ клиента' для 1С:Предприятие 8. 
Добавь обработку BeforeWrite для проверки наличия товара на складе.
Используй типичные паттерны 1С."
```

**Проводки:**
```
"Создай метод проведения для документа 'Поступление товаров'.
Необходимо сделать проводки по регистрам накопления 'ОстаткиТоваров'.
Добавь обработку взаимозачётов."
```

#### 2. Отчёты и обработки

```
"Создай отчёт в 1С, который показывает обороты по складам 
за выбранный период. Добавь группировку по номенклатуре 
и остаткам на начало/конец периода."
```

#### 3. SQL-запросы для встроенной БД

```
"Напиши запрос к временной таблице для анализа данных.
Используй GROUP BY, HAVING и оконные функции.
Оптимизируй для больших объёмов данных."
```

#### 4. Печатные формы (кейс GPT-5)

**Процесс:**
1. Создай макет печатной формы
2. Выгрузи в XML
3. Отдай XML ИИ с описанием требований
4. Получи готовый код

**Что должен учесть ИИ:**
- Перенос строк при выводе товаров
- Подвал на каждом листе (областью)
- Дублирование шапки таблицы на следующих листах
- Вывод других областей после таблицы

#### 5. Автоматизация рутины

**Скрипты для миграции данных:**
```
"Напиши обработку для переноса данных из одной конфигурации в другую.
Используй выгрузку/загрузку через XML.
Добавь проверку целостности данных."
```

**Утилиты для отладки:**
```
"Создай отладчик для отслеживания выполнения кода.
Добави логирование в файл с таймстампами."
```

#### 6. Интеграция 1С с внешним миром

**HTTP-сервисы (REST API):**
```
"Реализуй HTTP-сервис в 1С для получения списка номенклатуры.
Используй JSON для обмена данными.
Добавь авторизацию по токену."
```

**Обмен данными через JSON/XML:**
```
"Создай обработку для импорта данных из JSON.
Добавь валидацию и обработку ошибок."
```

### 🎭 Оркестрация моделей

**Проблема:** Один агент жрёт лимиты даже на Max-плане

**Решение:** Разные модели для разных задач

**Архитектура (пример от Pavel Stupko):**
- **Opus 4.5** — основной диалог (сложные задачи)
- **Sonnet 4.5** — рутинные задачи (дешевле)
- **Gemini 3** — управление памятью и изображениями

**Экономия:** 80-90% лимита Max-плана → оптимизация до Pro ($20)

**Как настроить:**
- В Cursor создай Custom Modes
- Для каждого mode укажи свою модель
- Используй разные modes для разных задач

### 🧠 Личность агента

### 🔍 RAG (Retrieval-Augmented Generation)

**Что это:** Подключение ИИ к неструктурированным данным компании

**Типы RAG:**
- **GraphRAG** — графовый поиск для связанных данных
- **CRAG** — критический RAG с проверкой фактов
- **LightRAG** — лёгкий RAG для быстрых ответов
- **RAPTOR** — иерархический RAG для больших данных

**Векторные БД:**
- **Milvus** — масштабируемая векторная БД
- **Qdrant** — векторная БД с фильтрацией
- **pgVector** — расширение для PostgreSQL
- **ChromaDB** — простой векторный движок

**NoSQL БД:**
- **Elastic** — полнотекстовый поиск
- **Neo4j** — графовая БД

**Инфраструктура:**
- **vLLM** — кластеризация локальных моделей
- **Embedding-модели** — векторизация текста
- **Chunk-и** — разбиение текста
- **Reranking** — переранжировка результатов
