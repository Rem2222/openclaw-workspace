# Материалы по вайбкодингу - Сбор

## Материал #1: Обновление cursor_rules_1c

**Источник:** https://github.com/comol/cursor_rules_1c

### Ключевые изменения

**11 субагентов (фокус модели, изоляция контекста, экономия токенов):**
1. Аналитик
2. Архитектор
3. Разработчик
4. Технический писатель
5. Фиксер ошибок
6. Планировщик
7. Рефакторинг
8. Тестировщик
9. Эксперт по производительности
10. Ревьювер кода
11. Ревьювер архитектуры

**Преимущества субагентов:**
- Фокус модели на конкретной задаче
- Изоляция контекста
- Экономия токенов и денег (разные модели для разных задач)
- Более качественные решения

**2 новых скилла:**
- БСП (Библиотека стандартных подсистем)
- Оптимизация запросов

**MCP фреймворки:**
- Поддержка: Openspec, spek-kit, memory-bank, taskmaster
- Рассмотрены на курсе: https://vibecoding1c.ru/vibecourse
- Корректная интеграция с правилами
- Свой SDD фреймворк не нужен (в готовых всё хорошо реализовано)
- Правила работают и БЕЗ SDD фреймворков

**Структура правил:**
- 11 файлов правил
- Только 2 обязательных (небольшие)
- Существенная экономия токенов
- Новые правила и паттерны на основе опыта
- Исправлены неточности

**Размещение:**
- Файлы в папке `.cursor` проекта
- Папку можно создать если нет
- На GitHub - бесплатно

**Особенности:**
- Субагент-оркестратор пока не добавлен (human-in-the-loop)
- Браузер для 1С скорее не нужен
- Ориентировано на MCP-серверы: https://vibecoding1c.ru/mcp_server
- Без MCP работает, но с меньшей эффективностью
- Автор использует пару дней, возможны косяки
- Совместимо с другими IDE (поддерживающими формат Claude)
- Для Claude Code переименовать папку `.cursor`

**Планы:**
- Добавить лекцию в оба курса по этим правилам
- Возможно вернуться к субагенту-оркестратору после генерации метаданных и форм

## Материал #2: Продукты vibecoding1c.ru

**Источник:** Telegram канал @comol_it_does_matter

### Полная линейка продуктов

#### 🆕 Продвинутый вайбкодинг для 1С
**Ссылка:** https://vibecoding1c.ru/vibecourse

**Цель:** Системная разработка в 1С с ИИ. Не про промпты, а про архитектуру и стабильный результат.

**Внутри:**
- SDD (Software Design Document)
- Spec-Kit, OpenSpec
- Cursor Memory Bank
- Разработка MCP-серверов
- Практические трюки Cursor и MCP

**Для кого:** 1С-разработчики, архитекторы, техлиды с опытом, понимающие основы LLM, контекста, RAG и Rules. Не для полного нуля.

**Формат:** Видео, практика, разборы

**Обновление (15.01.2026):**
- 10 лекций, транскрибация, исходники MCP и части OneAPA
- Бонус: учимся MCP разрабатывать
- Рассмотрение ИИ с EDT и Supercode для Cursor
- Митап по EDT/Supercode через 2 недели

**Для кого актуален:** Кто понял, что нужен SDD фреймворк или Memory Bank (ИИ теряет контекст, ломает проект, тратит кучу токенов)
