# Анализ бага в функции remove_minors()

## Как выглядел бы баговый код

```python
def remove_minors(users: list) -> list:
    for user in users:  # Итерация по оригинальному списку
        if user["age"] < 18:
            users.remove(user)  # Удаление во время итерации!
    return users
```

## В чём баг

При удалении элемента из списка во время итерации:

1. **Сдвиг индексов:** После `.remove()` все следующие элементы сдвигаются влево
2. **Пропуск элементов:** Итератор переходит к следующему индексу, но из-за сдвига пропускает элемент

### Пример

```python
users = [
    {"name": "Alice", "age": 30},   # индекс 0
    {"name": "Bob", "age": 15},     # индекс 1 ← удаляем
    {"name": "Diana", "age": 12},   # индекс 2
    {"name": "Charlie", "age": 35}, # индекс 3
]
```

**Шаги:**
- Индекс 0: Alice → не удаляем
- Индекс 1: Bob → удаляем `users.remove(Bob)`
- После удаления список:
  ```
  [Alice (0), Diana (1), Charlie (2)]  # Diana сдвинулась с 2 на 1!
  ```
- Итератор переходит к индексу 2 → Charlie
- **Diana (теперь индекс 1) пропущена!**

**Результат:** `['Alice', 'Diana', 'Charlie']` ← Diana должна быть удалена!

## Как исправить

### Способ 1: Копия для итерации
```python
for user in users[:]:  # users[:] создаёт копию
    if user["age"] < 18:
        users.remove(user)
```

### Способ 2: List comprehension (используется в файле)
```python
return [user for user in users if user["age"] >= 18]
```

**Почему list comprehension лучше:**
- Не модифицирует оригинальный список (pure function)
- Нет проблем с индексами
- Более читаемо и питонично
- Работает быстрее для больших списков
