# 2026-03-17 — Оркестратор проектов

## Задача:
Разработка системы оркестрации агентов для управления проектами

## Роли агентов:
1. **Аналитик требований** — общение с пользователем, сбор требований
2. **Системный аналитик** — декомпозиция на этапы, планирование
3. **Технический писатель** — написание ТЗ по этапам
4. **Ревьюер/Архитектор** — проверка ТЗ на ошибки/пробелы
5. **Разработчики** — frontend, backend, 1С, тесты
6. **Тестировщик** — тестирование результатов
7. **DevOps** — деплой и настройка окружения

## Структура процесса:
1. Аналитик требований → обсуждение → спецификация
2. Системный аналитик → декомпозиция → план разработки
3. Технический писатель → ТЗ по каждому этапу
4. Ревьюер → проверка ТЗ → исправления
5. Планировщик → определение параллелизуемых этапов
6. Запуск задач в субагентах согласно плану

## Передача контекста:
Через файлы с явным указанием документации/ссылок/файлов

## Контроль качества:
- Чек-лист к каждому этапу (на этапе ТЗ)
- Автотесты при необходимости
- Решение о переходе принимает пользователь

## Структура папки проекта:
```
~/.openclaw/workspace/projects/ПроектX/
├── README.md
├── ТЗ.md
├── План_разработки.md
├── Лог_общения.md        (полный лог)
├── Лог_проекта.md        (краткий лог)
├── Todo.md               (чек-лист этапов)
├── STATUS.md             (дашборд проекта)
├── Этап1_Аналитика/
├── Этап2_Разработка/
└── ...
```

## Обработка ошибок:
- Зависший агент → убить и перезапустить (до 3 раз)
- 3 сбоя подряд → аналитик ищет причину → решение пользователю
- Логировать: "завис/перезапуск"

## Именование субагентов:
`тип-агента-что-делает-этап-задача`  
Пример: `dev-frontend-main-page-2-1`

## Память:
- Только контекст от оркестратора
- Не удалять после завершения

## Git:
- Решается на этапе 1 (тип проекта)
- Не для мелких доработок

## Дашборд:
**ЗАДАЧА:** Разработать веб-морду для отображения:
- Активные проекты
- Текущий этап
- Прогресс (%)

## Статусы этапов:
- `pending` — ожидает запуска
- `in_progress` — выполняется
- `review` — на проверке
- `done` — завершено
- `failed` — ошибка
