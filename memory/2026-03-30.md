# AIChat EPF для 1С — продолжение

Продолжаем чинить AIChat EPF. Последние правки:

### JSON-парсер
- `ТипJSON.НачалоОбъекта` — **не существует** как системное перечисление в 1С 8.3.27. Ошибка: "Переменная не определена (ТипJSON)"
- Заменили кастомный JSON-парсер на платформенный `ПрочитатьJSON(Строка, Истина)` — это работает

### Получение тела HTTP-ответа
- `HTTPОтвет.ПолучитьТелоКакСтроку(КодировкаТекста.UTF8)` — **не принимает параметров** в этой версии 1С
- `ПолучитьСтрокуИзДвоичныхДанных(Данные, КодировкаТекста.UTF8)` — работает для конвертации binary → string
- Тип возврата `ПолучитьТелоКакСтроку()` непредсказуем — может вернуть Строка, ДвоичныеДанные, или Неопределено

### Текущая ошибка
- `ПрочитатьJSON(ОтветСтрока, Истина)` — "Несоответствие типов (парам 1)" на строке 150
- `ОтветСтрока` чем-то не устраивает `ПрочитатьJSON`
- Предполагаение: сервер возвращает сжатый (gzip) ответ, `ПолучитьТелоКакСтроку()` возвращает `ДвоичныеДанные`, `ПолучитьСтрокуИзДвоичныхДанных` не справляется
- Добавлена отладка: теперь в ошибке виден тип `ТелоОтвета`

### URL-парсинг (исправлено ранее)
- Все функции URL теперь используют локальные переменные вместо перезаписи параметра
- `ПолучитьПортИзURL` исправлен — не ссылался на глобальный `АдресAPI`

## GLM-5.1 HTTP 404 — статус

- OpenClaw TUI и webchat: glm-5.1 возвращает HTTP 404
- Direct curl к `https://api.z.ai/api/coding/paas/v4` работает для glm-5.1
- glm-4.7 работает в обоих
- `session_status` переключает модель временно (сбрасывается)
- Проблема не в конфиге и не в API — скорее всего что-то с маршрутизацией внутри OpenClaw для этой конкретной модели
