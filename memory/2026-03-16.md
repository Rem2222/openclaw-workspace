## Архитектура памяти OpenClaw

**Вопрос:** Память на двоих агентов общая или нет?

**Ответ:**
- **Долгосрочная память (MEMORY.md)** — ОБЩАЯ для всех агентов
- **Дневники (memory/*.md)** — ОБЩИЕ для всех агентов
- **История сессий** — ИЗОЛИРОВАННАЯ по агентам

**Структура:**
```
~/.openclaw/workspace/MEMORY.md ← ДОЛГОСРОЧНАЯ (ОБЩАЯ)
~/.openclaw/workspace/memory/*.md ← ДНЕВНИКИ (ОБЩИЕ)
~/.openclaw/agents/main/sessions/ ← СЕССИИ MAIN (ИЗОЛИРОВАННЫЕ)
~/.openclaw/agents/1c/sessions/ ← СЕССИИ 1C (ИЗОЛИРОВАННЫЕ)
```

**Вывод:** Агенты могут делиться знаниями через MEMORY.md, но их диалоги не перемешиваются.

**Проблема:** Агент "1c" не видит правила из `.claw/rules/` — они загружаются только для агента main.

## OpenViking от Volcengine

**Что:** Контекстная база данных для AI агентов с файловой парадигмой

**Ключевые фичи:**
- Файловая система вместо плоского RAG
- L0/L1/L2 уровни контекста (абстракция/обзор/полное)
- Рекурсивный поиск по директориям
- Наблюдаемый поиск (визуализация пути)

**Результаты LoCoMo10:**
- 52.08% задач (vs 35.65% у базового OpenClaw)
- 90% экономия токенов

**Установка:**
```bash
python3 -m venv ~/.openviking/venv
source ~/.openviking/venv/bin/activate
pip install openviking
openviking-server
```

**Конфиг:** `~/.openviking/ov.conf`

**Проблема:** Требует платные API (OpenRouter):
- Embedding: text-embedding-3-large (~$0.004/100k tokens)
- Overview: gpt-4o (~$2.50/1M tokens)

**Стоимость для GitHub репозитория:** ~$0.50-2.00

**Вердикт:** Отличная архитектура, но не готова для production без платных API.

**Рекомендация:** Оставить текущий стек OpenClaw (SQLite Vec + FTS5 + локальный embedding).

**Сервер запущен:** tmux session "openviking", порт 1933

**Ожидается решение:**

### Установка ollama:

```bash
# Исправление DNS в WSL2
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf

# Установка ollama
curl -fsSL https://ollama.com/install.sh | sh

# Загрузка моделей
ollama pull nomic-embed-text  # embedding (768 dims)
ollama pull qwen2.5:7b       # overview (или qwen3.5:27b)
```
