### OpenCode API исследование

- **Endpoint:** `POST https://api.opencode.ai/v1/workspaces`
- **Авторизация:** требует валидную сессию (куку). Без авторизации возвращает HTTP 200 "Not Found"
- **Оригинальный CodexBar-Win** использует `_CookieDecryptor` — читает куку `sessionKey` из Chrome cookies database (`%LOCALAPPDATA%\Google\Chrome\User Data\Default\Network\Cookies`), расшифровывает через Windows DPAPI + AES-GCM
- **Проблема:** у Рема в браузере для opencode.ai есть только `auth` и `oc_locale` куки, нет `sessionKey`
- **Возможные решения:**
  - OpenCode может использовать localStorage/sessionStorage вместо cookies
  - Кука `auth` может содержать токен в другом формате
  - Нужны дальнейшие исследования
