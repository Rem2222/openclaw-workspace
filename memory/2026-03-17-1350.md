## Проблемы и решения

### Проблема 1: Ошибка конфигурации
```
ValidationError: Extra inputs are not permitted [type=extra_forbidden]
```

**Решение**: Убрал секцию `providers` из `ov.conf` — она не поддерживается OpenViking

### Проблема 2: Неясные эндпоинты VikingBot

**Решение**: Использовать Console Web UI на http://localhost:18791 для настройки и тестирования

### Проблема 3: Блокировка векторной базы

**Ошибка:**
```
IO error: lock /home/rem/.openviking/workspace/vectordb/context/store/LOCK
```

**Причина:** VikingBot gateway удерживает эксклюзивную блокировку на базе данных, и скрипты не могут получить доступ к поиску.

**Решения:**
1. **Остановить VikingBot gateway** — освободит блокировку
2. **Использовать VikingBot как шлюз** — отправлять запросы через API
3. **Использовать общий клиент (Singleton)** — сложная синхронизация

**Рекомендация:**
- Для тестирования: Остановить gateway
- Для продакшена: Использовать VikingBot как шлюз

## Уроки

- **VikingBot — готовое решение**: Не нужно писать скилл с нуля, можно использовать OpenViking напрямую через Python API
- **Оllama медленный для L0/L1**: Модель `qwen2.5:1.5b` слишком медленная, нужно `qwen2.5:7b` или OpenRouter
- **Конфигурация чувствительна**: OpenViking строго валидирует `ov.conf`, лишние секции вызывают ошибки
- **Блокировка базы данных**: VikingBot gateway удерживает эксклюзивную блокировку, что мешает параллельным запросам
