## Lossless Claw (LCM) — Анализ

### Что это
Плагин для OpenClaw — LCM (Lossless Context Management), заменяет стандартную систему compaction.

### Как работает
1. Хранит ВСЕ сообщения в SQLite (`~/.openclaw/lcm.db`)
2. При превышении 75% контекста — сжимает старые в DAG-суммаризации
3. Инструменты: `lcm_grep`, `lcm_describe`, `lcm_expand`

### Установка
```bash
openclaw plugins install @martian-engineering/lossless-claw
```

### Конфигурация
```json
{
  "freshTailCount": 32,           // Последние 32 сообщения не сжимать
  "contextThreshold": 0.75,         // Сжимать при 75% контекста
  "incrementalMaxDepth": -1,        // Неограниченная глубина
  "summaryProvider": "anthropic",
  "summaryModel": "claude-3-5-haiku"
}
```

### Сравнение с OpenViking
| Характеристика | LCM | OpenViking |
|---|---|---|
| Тип | Плагин OpenClaw | Отдельная система |
| Интеграция | Встроенная | Внешняя |
| Хранилище | SQLite | PostgreSQL? |
| Структура | DAG суммаризаций | 4 уровня (L0-L3) |
| Управление | Автоматическое | Ручное |
| Установка | `openclaw plugins install` | Python venv + pip |

## Инцидент: Ошибка Gateway после удаления

### Проблема
```bash
(venv) rem@XE2690:~/.openclaw/skills/openviking-memory$ openclaw gateway restart
sh: 0: getcwd() failed: No such file or directory
```

### Причина
Пользователь был в директории `~/.openclaw/skills/openviking-memory/`, которая была удалена. Node.js не мог получить `cwd`.

### Решение
```bash
cd ~/.openclaw/workspace && openclaw gateway restart
```
